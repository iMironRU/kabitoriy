# tests/ — модульные тесты (YAxUnit)

Тест-расширение `tests` для конфигурации Кабиторий. Движок — **YAxUnit**
(см. `CLAUDE.md` → «Тестирование»).

## Структура

- `src/` — исходники расширения (выгрузка в XML).
  - `CommonModules/ОМ_ТестыОказаниеУслуги` — тесты ядра КА `ОказаниеУслуги`:
    допустимые/недопустимые переходы, обязательность Исхода при закрытии,
    ось блокировки (§6.2), форс сквозь блок, идемпотентность СнятьБлок.

Тестовые модули — контекст **Клиент(управляемое+обычное) + Сервер**; тесты
ядра регистрируются как серверные (`ЮТТесты.ДобавитьСерверныйТест`), т.к.
работают с транзакциями/объектами. В файловой ИБ толстый клиент исполняет
их в своём процессе.

## Запуск

Одной командой (полный флоу: загрузка конфы + расширений, снятие безопасного
режима, прогон, разбор отчёта):

```bash
bash scripts/run-tests.sh
```

Предварительно (один раз) положить в `tools/yaxunit/` (gitignored):
- `YAxUnit.cfe` — с [releases YAxUnit](https://github.com/bia-technologies/yaxunit/releases);
- `DisableSafeMode.epf` — из `tools/local-test/scripts/lib/` репозитория YAxUnit.

Платформа и dev-ИБ берутся из `.v8-project.json` (`v8path`, `base/ib_dev`).
Отчёт jUnit — `base/yax-report.xml`, код возврата — `base/yax-exit.txt`.

> Прогон требует **легальной лицензии** платформы 1С.
