# 1c/

Исходники 1С в двух формах развёртывания (konspekt §18):
- `extension/` — расширение, встраивается в хост-конфигурацию (читает её данные напрямую);
- `standalone/` — самостоятельное приложение на БСП (своя база, синхронизация обменом БСП).

Доменный код один на обе формы; различаются реализации адаптера «Источник»
(CLAUDE.md → «Слой источника данных»).

**Формат исходников (EDT-проект или выгрузка конфигурации в XML) ещё не выбран.**
Решаем перед первым коммитом кода: от этого зависят `.gitignore` и сборка/хранилище.
