## Язык, кодировка, типографика

- Код, комментарии, синонимы метаданных, сообщения пользователю — **русский (кириллица)**.
- Буква **«ё» не используется** в коде и метаданных.
- Тире `—` **не использовать**, заменять дефисом `-` (проблемы кодировки в платформе). Кавычки для пользовательских текстов — `«...»`. Неразрывные пробелы и невидимые Unicode-символы запрещены.
- Английский допустим только для: имён веб-сервисов и их методов, параметров HTTP-запросов, полей JSON для внешних API.

## Именование

### Переменные (PascalCase)

Имена отражают **бизнес-смысл**, не тип. Венгерская нотация запрещена.

```bsl
// ПРАВИЛЬНО
Заявки
ТекущаяЗаявка
СоответствиеСегментов

// НЕПРАВИЛЬНО — венгерская нотация / сокращения
МассивЗаявок     // → Заявки
табЗаяв          // → Заявки
_ИмяПеременной   // подчёркивание в начале

// ЗАПРЕЩЕНО — имена глобального контекста как локальные переменные
Документы, Справочники, Регистры, Метаданные, Константы, Пользователи
```

### Булевы — только положительные имена (имя отражает состояние `Истина`)

```bsl
ЕстьОшибки        // не НетОшибок
ЗаявкаПодана      // не ЗаявкаНеПодана
ПроверкаПройдена  // не ПроверкаНеПройдена
```

### Процедуры и функции

| Назначение | Паттерн | Пример |
|------------|---------|--------|
| Возвращает описание | Существительное | `ПолноеИмя()` |
| Создаёт объект | Префикс «Новый» | `НовоеНамерениеOutboxа()` |
| Проверяет условие | «Это»/причастие | `ПереходДопустим()`, `ЗаявкаПодана()` |
| Выполняет действие | Глагол-инфинитив | `ВыполнитьПереход()` |
| Получает данные | Префикс «Получить» | `ПолучитьСнимокПоложения()` |
| Устанавливает | Префикс «Установить» | `УстановитьБлок()` |
| Конструктор параметров | Префикс «Параметры» | `ПараметрыПерехода()` |

- Префикс `Получить*` **не нужен**, когда результат очевиден из имени или возвращается коллекция (`Заявки()`, не `ПолучитьЗаявки()`). Допустим при делегировании к платформе/контракту БСП.
- **Описание `Экспорт`-методов обязательно**, начинается с глагола («Возвращает…», «Проверяет…»), не с «Процедура…»/имени функции.
- **Экспортные функции** именуются с префиксом `Кабиторий_`; внутренние (не экспорт) — без префикса.

### Метаданные

| Элемент | Правило |
|---------|---------|
| Новые объекты/роли/реквизиты | Префикс `Кабиторий_` |
| Подсистемы-маркеры (при модификации БСП) | `Кабиторий_ДобавленныеОбъекты`, `Кабиторий_ИзмененныеОбъекты` |
| Реквизиты внутри собственных объектов | **Без префикса** (имя объекта уже его несёт) |
| Перечисления строковых значений | В алфавитном порядке |

### Строки и числа

```bsl
// СтрШаблон вместо конкатенации
Сообщение = СтрШаблон("Заявка %1 переведена в %2", Номер, НовоеСостояние);

// Магические числа — запрещены, выносить в именованную переменную/константу
МаксимумПопытокOutboxа = 5;
Если Попыток > МаксимумПопытокOutboxа Тогда
```

## Структура модулей (области)

**Общий модуль:**

```bsl
#Область ПрограммныйИнтерфейс
// Экспортное публичное API (с префиксом Кабиторий_)
#КонецОбласти

#Область СлужебныйПрограммныйИнтерфейс
// Экспорт для других модулей расширения/конфигурации
#КонецОбласти

#Область СлужебныеПроцедурыИФункции
// Внутренняя реализация (не экспорт)
#КонецОбласти
```

**Модуль объекта (документ Заявка и т.п.):** `ОписаниеПеременных` → `ОбработчикиСобытий` → `СлужебныеПроцедурыИФункции` → `Инициализация`.

**Модуль формы:** `ОписаниеПеременных` → `ОбработчикиСобытийФормы` → `ОбработчикиСобытийЭлементовШапкиФормы` → `ОбработчикиСобытийЭлементовТаблицыФормы` → `ОбработчикиКомандФормы` → `СлужебныеПроцедурыИФункции` (с разделением по `&НаКлиенте`/`&НаСервере`/`&НаСервереБезКонтекста`).

Псевдообласти через комментарии запрещены — только `#Область`/`#КонецОбласти`.

## Директивы компиляции

| Операция | Директива |
|----------|-----------|
| Массовые операции с данными, чтение для отображения | `&НаСервереБезКонтекста` |
| Изменение реквизитов формы | `&НаСервере` |
| Интерактив | `&НаКлиенте` |
| Асинхронные операции | `&НаКлиенте` + `Асинх` |

**Правило предпочтения:** `&НаСервереБезКонтекста` > `&НаСервере`. `&НаСервере` — только когда реально нужен контекст формы.

## Каталог анти-паттернов

Приоритет = Влияние × Частота × Объём данных.

### Critical

**1. Запрос в цикле** — O(n) обращений к БД вместо O(1).
```bsl
// ❌
Для Каждого Строка Из Таблица Цикл
    Запрос.УстановитьПараметр("Ссылка", Строка.Объект);
    Запрос.Выполнить();
КонецЦикла;
// ✅ — один пакетный запрос
Запрос.УстановитьПараметр("СписокСсылок", Таблица.ВыгрузитьКолонку("Объект"));
Запрос.Текст = "... ГДЕ Ссылка В (&СписокСсылок)";
```

**2. Обращение к реквизитам ссылки через точку** — грузит весь объект.
```bsl
// ❌  Имя = Контрагент.Наименование;
// ✅  Имя = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(Контрагент, "Наименование");
//     Рекв = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(Контрагент, "Наименование, ИНН");
```

**3. Подзапрос в SELECT** — N+1. Заменять соединением с агрегацией (`ЛЕВОЕ СОЕДИНЕНИЕ … СГРУППИРОВАТЬ ПО`).

### High

**4. Фильтрация виртуальной таблицы в WHERE** вместо параметров ВТ: `…Остатки(, Склад = &Склад)`, не `…Остатки() ГДЕ Склад = &Склад`.
**5. Отсутствие `ПЕРВЫЕ N`** на потенциально больших выборках.
**6. Избыточные клиент-серверные вызовы** — консолидировать в один серверный вызов, возвращающий Структуру.
**7. `&НаСервере` вместо `&НаСервереБезКонтекста`** — лишняя передача контекста формы.

### Medium

**8. Отсутствие кеширования в цикле** — повторные дорогие вызовы для одинаковых ключей; кешировать через `Соответствие`.
**9. Алгоритм O(n²)** (вложенные циклы поиска) — заменять индексом через `Соответствие` → O(n).
**10. Глубокая вложенность** (>3-4 уровней) — ранний выход (`Если НЕ Условие Тогда Возврат;`).

### Оптимизация запросов

| # | Анти-паттерн | Правильно |
|---|--------------|-----------|
| 16 | Подзапрос в JOIN | Временная таблица `ПОМЕСТИТЬ ВТ_…` |
| 17 | Соединение с виртуальной таблицей напрямую | Сначала ВТ через `ПОМЕСТИТЬ` |
| 18 | Множественные `ИЛИ` в WHERE | `В (&Список)` или `ОБЪЕДИНИТЬ ВСЕ` |
| 19 | Разыменование составного типа | `ВЫРАЗИТЬ(Поле КАК Справочник.X).Реквизит` |
| 20 | Нет индекса по фильтруемым полям | Индексировать поля фильтра/соединения |
| 21 | Цепочка `Запрос.Выполнить().Выбрать()` | Промежуточная переменная |
| 22 | ВТ без префикса | Префикс `ВТ_` |
| 23 | Конкатенация строк в тексте запроса | Только `УстановитьПараметр()` (инъекция!) |

### Архитектурные

- **God Module** (>2000 строк) → декомпозиция по ответственности.
- **Tight Coupling** → публичный API + параметры, программные интерфейсы БСП.
- **Copy-Paste** → выделение общих функций, DRY.
- **Big Ball of Mud** → структура модулей с областями.
- **Premature Optimization** → сначала работающий код, затем профилирование, затем оптимизация по данным.

### Правила написания запросов

1. Текст запроса — с новой строки, не в строке с `Запрос.Текст =`.
2. Результат — всегда в промежуточную переменную (цепочки запрещены).
3. Псевдонимы через `КАК` — для всех полей.
4. Параметры — только `УстановитьПараметр()`.
5. ВТ — префикс `ВТ_`.
6. Предпочитать `ВНУТРЕННЕЕ СОЕДИНЕНИЕ` над `ЛЕВОЕ`, где допустимо.
7. Измерения регистров — фильтровать первыми.
8. `ПЕРВЫЕ N` для ограниченных выборок; индексы — на поля фильтрации/соединения.

## Обработка ошибок

> ⚠️ **Транзакции, внешние вызовы, переходы — по `CLAUDE.md`** (короткие транзакции, Outbox вне транзакции, защищённые переходы). Ниже — общий стиль, не отменяющий те MUST-правила.

- **Не** использовать `Попытка…Исключение` для управления потоком и штатных проверок — предварять проверкой условия.
- **Не подавлять** исключения без обработки (пустой блок `Исключение` запрещён).
- Значимые ошибки — логировать с контекстом (заявка, переход, исполнитель, `idempotencyKey`/`externalRef`/`messageId` — см. `CLAUDE.md` «структурное журналирование»).
- Паттерн интеграционного метода: возврат Структуры `{Успех, Данные, Ошибка}`; валидация без исключений; бизнес-логика в транзакции с `ОтменитьТранзакцию()` в `Исключение`.

```bsl
// Проверка ПЕРЕД действием — не исключение для контроля потока
Если Заявка.Проведена Тогда
    Возврат;
КонецЕсли;
НачатьТранзакцию();
Попытка
    // короткие операции с данными 1С (без внешних вызовов!)
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    ВызватьИсключение;
КонецПопытки;
```

## Комментарии

**Код самодокументируем.** Комментарий объясняет **ПОЧЕМУ**, не **ЧТО**. Если нужен, чтобы объяснить *что* делает код — рефакторинг (разбить, переименовать, упростить).

**Допустимо:** бизнес-правила; workaround'ы (со ссылкой на баг/версию); предупреждения о контексте («вызывается из фонового задания»); пояснение сложного алгоритма; `TODO: <ID-задачи> …`.

**Документирование `Экспорт`-методов** — обязательно (Параметры / Возвращаемое значение в стандартном формате БСП).

## Запрещённые практики

| Практика | Альтернатива |
|----------|--------------|
| `Сообщить()` | `ОбщегоНазначения.СообщитьПользователю()` / `…Клиент.СообщитьПользователю()` |
| `Выполнить()` / `Вычислить()` | Явная логика (безопасность) |
| Тернарный `?(…)` | `Если…Иначе` или функция |
| `ТекущаяДата()` на сервере | `ТекущаяДатаСеанса()` (для SLA — ещё и пояс Центра, `konspekt §14`) |
| Венгерская нотация / глобальные имена как переменные | Бизнес-имена |
| Пустые исключения | Логирование + обработка |
| Сравнение с `Истина`/`Ложь`, Yoda-синтаксис | Булево выражение напрямую (`Если Сумма = 0`) |
| Хардкод учётных данных | БСП «Безопасное хранение паролей» / защищённые константы |
| Параметр функции как доп. выход | Возврат через возвращаемое значение / Структуру |
| COM-объекты, хардкод разделителя пути `\` | Кросс-платформенные функции, `/` |

**Платформенное:** `ЗначениеЗаполнено()` вместо прямых сравнений с пустыми ссылками; длительные операции (>10 c) — в фоновые задания (`ДлительныеОперации.ВыполнитьФункцию()`); `УстановитьПривилегированныйРежим()` — парно, только по необходимости, с учётом RLS; перед использованием метода платформы — проверить, что он есть в 8.3.24.

## Чек-лист ревью (BSL)

- [ ] Нет запросов в циклах
- [ ] Нет обращений через точку к реквизитам ссылок
- [ ] Нет подзапросов в SELECT; ВТ вместо подзапросов в JOIN
- [ ] Виртуальные таблицы фильтруются параметрами; есть `ПЕРВЫЕ N` где применимо
- [ ] Серверные вызовы минимизированы; `&НаСервереБезКонтекста` где возможно
- [ ] Нет O(n²); вложенность ≤ 4; нет цепочек вызовов
- [ ] ВТ с префиксом `ВТ_`; нет конкатенации строк в запросах
- [ ] Нет магических чисел / венгерской нотации / запрещённых практик
- [ ] Комментарии объясняют «почему»; нет мёртвого/закомментированного кода
- [ ] **Не нарушены MUST-инварианты `CLAUDE.md`** (переходы, Outbox, блокировки, SPI)
- [ ] Проходит bsl-language-server (Error = 0)
